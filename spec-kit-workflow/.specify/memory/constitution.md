# Todo List 应用 - 项目宪法

## 核心原则

### 1. 用户体验优先
- 简洁直观的界面设计
- 流畅的交互体验
- 即时的视觉反馈

### 2. 代码质量
- 清晰的代码结构
- 充分的注释说明
- 模块化的组件设计

### 3. 现代技术栈
- 使用最新稳定版本的技术
- 响应式设计支持多端
- 性能优化优先

### 4. 可维护性
- 可读性强的代码
- 一致的编码风格
- 易于扩展的架构

## 技术偏好

- **前端框架**: React + TypeScript
- **样式方案**: Tailwind CSS
- **状态管理**: React Hooks (useState, useContext)
- **存储**: LocalStorage (简单持久化)
- **构建工具**: Vite

## 成功标准

- [ ] 用户可以添加、删除、标记完成任务
- [ ] 支持任务分类/标签
- [ ] 响应式设计，支持移动端
- [ ] 流畅的动画效果
- [ ] 代码结构清晰，易于理解
